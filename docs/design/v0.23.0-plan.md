## v0.22.0 收口后的威胁模型现状

| 向量 | 状态 |
|---|---|
| V1 绕过 passkey | ✅ 已防（TEE 内验，C-1） |

**v0.23.0 的核心问题：把 V5 从「半信」推到「真信」，并把已建的 attestation/anti-replay 落到「默认强制」。**

## 待办归拢（按依赖/优先级）

### 🅰 主线候选（二选一定主题，见末尾）

**B 线 / R-1 — NXP 根锚定（#37 Phase 2 · #48）** — 最大 V5 增益
- 现状：attestation key 设备自签、无 NXP 证书链（TOFU）。
- 阻塞：**EL2GO 账号 + IMX93SRM(NDA)**，已申请、pending。
- 拿到资料后：B1 ELE 内生成 attestation key + `hsm_pub_key_attest` 出证书；B2 evidence 携带 secure-boot 度量；B3 verifier 补 NXP 链验。
- 性质：**外部 gated**，资料到位即可做。

**D1 — attestation 默认绑进 SignHash / WebAuthn ceremony** — 不卡外部
- 现状：attestation 是可选 `/attestation` 端点，客户端可跳过 → V5 缓解停留理论。
- 做：每次签名响应自带 attestation（默认发生）。
- 前置：定 **per-sign vs session** 设计决策（per-sign 有 RSA-PSS 延迟/体积成本）+ SDK 配合。
- 性质：纯软件可做，需小设计 + 跨仓（SDK）协调。

### 🅱 强制化 / 硬件（高价值，各有 gate）

- **#50 RPMB key 生产编程** — 硬件 anti-rollback，解锁 R-9 TA 侧 + 去掉 `ree-fs-only` workaround。⚠️ **一次性不可逆熔丝**，需硬件定版 + 备份 + 明确授权。
- **#63 strict 生产 flip** — `ENFORCE_TA_CHALLENGE` 翻强制（关闭 legacy 重放后门，真正闭 V2）。**依赖 SDK #58 先发**（否则旧客户端全被拒），跨仓协调。
- **R-9 — attestation key 吊销/轮换** — 客户端侧已做（manifest sequence 防降级）；TA 侧绑 RPMB 单调计数器防回滚复活，**依赖 #50**。

### 🅲 捎带 / 机会项

- **#52 — TA 回传 signer 地址** — 已标记攒到下次 TA 改动；若 v0.23.0 动 TA（如 D1/#63），顺带做，去掉 gtoken 的「先 DeriveAddress」前置。
- **D2 — PSA Attestation Token（EAT/RFC 9783, COSE）** — 标准化 + Veraison 可自部署 verifier；需先实测 BSP 是否真暴露 EAT 接口。

### 🅳 靠后 / 按需

- **D3 — 链上 attestation verifier 合约** — 让 SuperPaymaster/SuperRelay 链上确认「KMS 跑真 TEE」。
- **#40 — CAAM 加速** — ⚠️ 实测 ELE/CAAM **不支持 secp256k1**，范围已缩；价值待重评。
- **#38 — PKCS#11 接口层** — 标准 HSM 接口，集成便利，非安全关键。

### 🅴 不是我们（生态跟踪，AirAccount #70 已闭）

- DVT 跨仓：airaccount-contract #110（链上验组合签名 + 强制大额需 DVT）/ aastar-sdk #63（聚合 + 打包）/ Brood #3（激励）+ SuperPaymaster 部署 PolicyRegistry。

## v0.23.0 主题建议（取决于 NDA）

- **若 NXP 一手资料到位** → 主题 **「硬件信任根锚定」**：B 线（R-1 + B1/B2/B3）把 V5 推到「不信任部署方」。这是质变。
- **若 NDA 仍 pending** → 主题 **「把 MVP 用满」**：**D1（attestation 默认化）+ #50（RPMB 防回滚，需授权）+ #63 strict flip（待 SDK #58）+ R-9 TA 侧**，把已建的 attestation/anti-replay 从「可用」推到「默认强制」。纯内部 + 跨仓协调，不卡 NXP。

> 建议：先确认 NDA 进度再定主题。在等期间，**D1 的设计（per-sign vs session）可以先出**，不浪费窗口。
